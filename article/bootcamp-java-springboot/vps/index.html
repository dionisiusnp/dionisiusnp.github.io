🧱 A. Setup dari 0 (VPS Kosong)

1. Install Docker

sudo apt update
sudo apt install docker.io -y
sudo systemctl enable docker
sudo systemctl start docker
---

2. Install Docker Compose

sudo apt install docker-compose -y
---

3. Setup MySQL (Docker)

mkdir -p ~/apps/mysql
cd ~/apps/mysql
nano docker-compose.yml

version: "3.8"
services:
  mysql:
    image: mysql:8
    container_name: mysql_server
    restart: always
    environment:
      MYSQL_ROOT_PASSWORD: root123
      MYSQL_DATABASE: spring_db
    ports:
      - "3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql

volumes:
  mysql_data:
Jalankan: 
docker compose up -d
---

4. Install Java

sudo apt install openjdk-17-jdk -y
java -version
---

5. Upload Spring Boot

mkdir -p ~/apps/springboot
scp app.jar user@server:~/apps/springboot/
---

6. Config aplikasi

spring.datasource.url=jdbc:mysql://127.0.0.1:3306/spring_db
spring.datasource.username=root
spring.datasource.password=root123
server.port=8007
---

7. Jalankan aplikasi

cd ~/apps/springboot
java -jar app.jar
Verifikasi: 
curl http://localhost:8007
---

8. Auto run (systemd)

sudo nano /etc/systemd/system/springboot.service

[Service]
User=ubuntu
WorkingDirectory=/home/ubuntu/apps/springboot
ExecStart=/usr/bin/java -jar app.jar
Restart=always

sudo systemctl enable springboot
sudo systemctl start springboot
---

🔥 B. Setup VPS dengan PRM (Docker sudah ada)

Kondisi:
  • Docker sudah terinstall
  • MySQL sudah ada (container)
  • Nginx proxy sudah ada
---

1. Upload Spring Boot

mkdir -p ~/apps/springboot
scp -r ~/Sites/spring-snapshot/* user@server:~/apps/springboot/
---

2. Jalankan aplikasi

java -jar app.jar
---

3. Problem utama

Spring Boot (host) tidak bisa akses MySQL (docker)
---

4. Solusi (Expose MySQL)

Edit docker-compose Laravel: 
ports:
  - "3306:3306"
Restart: 
docker compose down
docker compose up -d
---

5. Config Spring Boot

spring.datasource.url=jdbc:mysql://127.0.0.1:3306/spring_db
---

6. Integrasi Nginx Proxy

location / {
    proxy_pass http://103.196.153.149:8007;
}

🛠 Troubleshooting

DB tidak connect

mysql -h 127.0.0.1 -u root -p
---

Port tidak terbuka

ss -lnt
---

Connection refused

curl http://localhost:8007
---

Nginx error upstream

docker logs nginx_proxy
---

Debug Spring Boot

java -jar app.jar --debug
---

💡 Best Practice

✔ Gunakan systemd ✔ Pisahkan DB tiap aplikasi ✔ Gunakan reverse proxy ✔ Gunakan port berbeda tiap service ✔ Gunakan Docker untuk scaling
