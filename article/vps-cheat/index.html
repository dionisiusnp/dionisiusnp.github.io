VPS Deploy Cheat Sheet

Command yang sering dipakai saat setup VPS baru dan deploy aplikasi — dari initial setup Ubuntu hingga Nginx, PHP, MySQL, SSL, dan supervisor queue. Klik kode untuk copy.

🖥️
Initial Setup
💡 Langkah pertama setelah dapat akses root ke VPS baru. Buat user deploy terpisah supaya tidak pakai root sehari-hari.
Koneksi SSH ke VPS
ssh root@IP_VPS
ssh -i ~/.ssh/id_rsa root@IP_VPS   # pakai key file
Update & upgrade sistem
apt update && apt upgrade -y
apt install -y curl wget git unzip zip nano htop
Buat user deploy (non-root)
adduser deploy
usermod -aG sudo deploy

# Copy authorized_keys dari root ke user baru
rsync --archive --chown=deploy:deploy ~/.ssh /home/deploy
Setup SSH key di local machine
# Generate key (di local, bukan di VPS)
ssh-keygen -t ed25519 -C "email@example.com"

# Copy public key ke VPS
ssh-copy-id deploy@IP_VPS

# Atau manual
cat ~/.ssh/id_ed25519.pub | ssh root@IP_VPS "cat >> ~/.ssh/authorized_keys"
Disable root login & password auth (opsional, direkomendasikan)
# Edit /etc/ssh/sshd_config
nano /etc/ssh/sshd_config

  PermitRootLogin no
  PasswordAuthentication no

systemctl restart sshd
Set timezone
timedatectl set-timezone Asia/Jakarta
timedatectl status
🛡️
Firewall — UFW
⚠️ Pastikan allow SSH sebelum enable UFW, supaya tidak terkunci dari server.
Setup dasar
ufw allow OpenSSH              # atau: ufw allow 22
ufw allow 80                   # HTTP
ufw allow 443                  # HTTPS
ufw enable
ufw status verbose
Rule tambahan
ufw allow 'Nginx Full'          # allow port 80 & 443 via Nginx profile
ufw allow from IP_TERTENTU      # whitelist IP spesifik
ufw deny 3306                  # blokir akses MySQL dari luar
ufw delete allow 80            # hapus rule
ufw disable                    # matikan sementara
ufw reset                      # reset semua rule
🌐
Nginx
Install & manage service
apt install -y nginx

systemctl start nginx
systemctl enable nginx         # auto-start on boot
systemctl status nginx
systemctl reload nginx         # reload config tanpa downtime
systemctl restart nginx
Config server block untuk Laravel
nano /etc/nginx/sites-available/namaapp

----- isi file -----
server {
    listen 80;
    server_name domain.com www.domain.com;
    root /var/www/namaapp/public;
    index index.php index.html;

    add_header X-Frame-Options "SAMEORIGIN";
    add_header X-Content-Type-Options "nosniff";

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        fastcgi_pass unix:/var/run/php/php8.3-fpm.sock;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include fastcgi_params;
    }

    location ~ /\.(?!well-known).* {
        deny all;
    }

    client_max_body_size 50M;
}
Aktifkan site & test config
ln -s /etc/nginx/sites-available/namaapp /etc/nginx/sites-enabled/

nginx -t                        # test konfigurasi
systemctl reload nginx
Hapus site / default
rm /etc/nginx/sites-enabled/default
rm /etc/nginx/sites-enabled/namaapp
systemctl reload nginx
Cek log Nginx
tail -f /var/log/nginx/error.log
tail -f /var/log/nginx/access.log

PHP & PHP-FPM ondrej/php PPA
Tambah PPA & install PHP 8.3
add-apt-repository ppa:ondrej/php -y
apt update

apt install -y php8.3 php8.3-fpm php8.3-cli \
  php8.3-mysql php8.3-pgsql php8.3-sqlite3 \
  php8.3-curl php8.3-gd php8.3-imagick \
  php8.3-mbstring php8.3-xml php8.3-zip \
  php8.3-bcmath php8.3-intl php8.3-redis
Manage PHP-FPM
systemctl start php8.3-fpm
systemctl enable php8.3-fpm
systemctl status php8.3-fpm
systemctl restart php8.3-fpm
Cek versi & ekstensi aktif
php -v
php -m                           # list ekstensi aktif
php -r "echo phpinfo();" | grep redis
Atur php.ini (upload, memory)
nano /etc/php/8.3/fpm/php.ini

  upload_max_filesize = 50M
  post_max_size       = 50M
  memory_limit        = 256M
  max_execution_time  = 120

systemctl restart php8.3-fpm
Switch versi PHP (jika multi-versi)
update-alternatives --config php
update-alternatives --set php /usr/bin/php8.3

MySQL / MariaDB
Install MySQL
apt install -y mysql-server
systemctl enable mysql
mysql_secure_installation    # wizard: set root pw, hapus test DB
Install MariaDB (alternatif)
apt install -y mariadb-server mariadb-client
systemctl enable mariadb
mysql_secure_installation
Buat database & user
mysql -u root -p

-- Di dalam MySQL prompt:
CREATE DATABASE nama_db CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'db_user'@'localhost' IDENTIFIED BY 'password_kuat';
GRANT ALL PRIVILEGES ON nama_db.* TO 'db_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;
Backup & restore database
# Backup
mysqldump -u root -p nama_db > backup.sql
mysqldump -u root -p --all-databases > all-db.sql

# Restore
mysql -u root -p nama_db < backup.sql
Cek status & log
systemctl status mysql
tail -f /var/log/mysql/error.log
⚡
Redis
Install & manage
apt install -y redis-server
systemctl enable redis-server
systemctl status redis-server

# Test koneksi
redis-cli ping              # harus reply: PONG
redis-cli info server
Atur password Redis (opsional)
nano /etc/redis/redis.conf

  requirepass password_redis_kuat
  bind 127.0.0.1            # hanya listen lokal

systemctl restart redis-server
Konfigurasi .env Laravel
REDIS_HOST=127.0.0.1
REDIS_PASSWORD=password_redis_kuat
REDIS_PORT=6379

CACHE_STORE=redis
SESSION_DRIVER=redis
QUEUE_CONNECTION=redis

Composer
Install Composer secara global
curl -sS https://getcomposer.org/installer -o composer-setup.php
php composer-setup.php
mv composer.phar /usr/local/bin/composer
rm composer-setup.php

composer --version
Update Composer
composer self-update
composer self-update --2      # paksa ke v2

Node.js / NVM
Install NVM (Node Version Manager)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# Reload shell
source ~/.bashrc

nvm --version
Install & manage Node.js via NVM
nvm install --lts             # versi LTS terbaru
nvm install 20
nvm use 20
nvm alias default 20          # set default
nvm ls                         # list versi terinstall

node -v
npm -v
Install Node.js via NodeSource (alternatif, system-wide)
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs

Laravel Deploy
✅ Urutan lengkap dari clone hingga aplikasi siap jalan di VPS.
Clone repo & setup direktori
cd /var/www
git clone git@github.com:username/nama-repo.git namaapp
cd namaapp
Install dependencies & setup .env
composer install --optimize-autoloader --no-dev

cp .env.example .env
nano .env                      # isi DB, APP_URL, dll

php artisan key:generate
Permission storage & bootstrap
chown -R www-data:www-data /var/www/namaapp
chmod -R 755 /var/www/namaapp
chmod -R 775 /var/www/namaapp/storage
chmod -R 775 /var/www/namaapp/bootstrap/cache
Migrate, seed & optimize
php artisan migrate --force
php artisan db:seed --force   # jika ada initial seed

php artisan storage:link
php artisan optimize
php artisan config:cache
php artisan route:cache
php artisan view:cache
Build frontend assets
npm ci
npm run build
Deploy ulang (setelah git pull)
cd /var/www/namaapp
git pull origin main

composer install --optimize-autoloader --no-dev
npm ci && npm run build

php artisan down
php artisan migrate --force
php artisan optimize:clear
php artisan optimize
php artisan queue:restart
php artisan up
Cek log Laravel
tail -f /var/www/namaapp/storage/logs/laravel.log

SSL — Let's Encrypt / Certbot
💡 Certbot otomatis modifikasi Nginx config dan tambahkan redirect HTTP → HTTPS.
Install Certbot
apt install -y certbot python3-certbot-nginx
Issue & install sertifikat
# Otomatis konfigurasi Nginx
certbot --nginx -d domain.com -d www.domain.com

# Hanya generate cert, config manual
certbot certonly --nginx -d domain.com
Renew & test auto-renew
certbot renew --dry-run        # simulasi renew
certbot renew                   # renew manual
certbot certificates            # list sertifikat aktif

# Auto-renew sudah diatur via systemd timer — cek dengan:
systemctl status certbot.timer
Hapus sertifikat
certbot delete --cert-name domain.com
👷
Supervisor — Queue Worker
💡 Supervisor menjaga proses queue:work tetap berjalan di background dan otomatis restart jika crash.
Install Supervisor
apt install -y supervisor
systemctl enable supervisor
systemctl start supervisor
Buat config worker Laravel
nano /etc/supervisor/conf.d/namaapp-worker.conf

----- isi file -----
[program:namaapp-worker]
process_name=%(program_name)s_%(process_num)02d
command=php /var/www/namaapp/artisan queue:work redis --sleep=3 --tries=3 --max-time=3600
autostart=true
autorestart=true
stopasgroup=true
killasgroup=true
user=www-data
numprocs=2
redirect_stderr=true
stdout_logfile=/var/www/namaapp/storage/logs/worker.log
stopwaitsecs=3600
Reload & manage Supervisor
supervisorctl reread
supervisorctl update
supervisorctl start namaapp-worker:*

supervisorctl status
supervisorctl restart namaapp-worker:*
supervisorctl stop namaapp-worker:*
Restart worker setelah deploy
# Via artisan (trigger graceful restart)
php artisan queue:restart

# Via supervisorctl (force restart)
supervisorctl restart namaapp-worker:*
⏰
Cron — Laravel Scheduler
Tambah cron entry untuk Laravel
# Edit crontab user www-data
crontab -u www-data -e

----- tambahkan baris ini -----
* * * * * cd /var/www/namaapp && php artisan schedule:run >> /dev/null 2>&1
Cek & manage cron
crontab -u www-data -l       # lihat cron aktif
crontab -u www-data -r       # hapus semua cron user ini

# Test scheduler berjalan
php artisan schedule:list
php artisan schedule:run       # jalankan sekali manual
Log output cron ke file
* * * * * cd /var/www/namaapp && php artisan schedule:run >> /var/www/namaapp/storage/logs/scheduler.log 2>&1
