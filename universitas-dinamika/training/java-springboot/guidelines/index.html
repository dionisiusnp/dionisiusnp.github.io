Struktur Project

Gunakan struktur package yang konsisten sejak awal. Ini akan memudahkan saat project berkembang.

src/main/java/com/example/app/ ├── config/ # SecurityConfig, WebConfig, dll ├── controller/ # REST Controllers (@RestController) ├── dto/ # Request & Response DTO │ ├── request/ │ └── response/ ├── entity/ # JPA Entities (@Entity) ├── exception/ # Custom exceptions & handler ├── repository/ # JPA Repositories ├── service/ # Business logic │ └── impl/ # Service implementations ├── security/ # JWT filter, UserDetails └── Application.java
📌 Jangan campur logic bisnis di Controller. Controller hanya menerima request, memanggil Service, dan mengembalikan response.

Lab 1 — Java Modern (OOP + Streams + Lambdas)

Sebelum Spring, pastikan konsep Java modern dikuasai. Spring Boot memanfaatkan Lambdas, Streams, dan Optional secara intensif.

// Tujuan Lab
  • Memahami OOP: Inheritance, Interface, Abstract Class, Generics
  • Menggunakan Lambda dan Functional Interface
  • Mengolah data dengan Stream API (filter, map, collect)
  • Menghindari NullPointerException dengan Optional
  • Menulis Record class untuk data sederhana (Java 16+)
LambdaAndStream.java
java
import java.util.*;
import java.util.stream.*;

public class LambdaAndStream {
    public static void main(String[] args) {

        List<String> names = List.of("Andi", "Budi", "Citra", "Dian");

        // Filter + map + collect
        List<String> result = names.stream()
            .filter(n -> n.startsWith("C"))
            .map(String::toUpperCase)
            .collect(Collectors.toList());
        // → ["CITRA"]

        // Optional — hindari null check manual
        Optional<String> first = names.stream()
            .filter(n -> n.startsWith("Z"))
            .findFirst();

        String val = first.orElse("Tidak ditemukan");
        first.ifPresent(n -> System.out.println(n));

        // Grouping
        Map<Integer, List<String>> byLength = names.stream()
            .collect(Collectors.groupingBy(String::length));
    }
}

// Record class (Java 16+) — immutable data carrier
public record UserRecord(Long id, String name, String email) {}
🧪 Latihan Mandiri

  2. Buat interface Greeting dengan satu method greet(String name). Implementasikan menggunakan lambda.
  3. Gunakan Optional untuk menangani pencarian user berdasarkan email dari sebuah List — kembalikan default jika tidak ada.

Lab 2 — Spring Core & Dependency Injection

Inti dari Spring adalah IoC Container. Memahami cara kerja Bean dan DI akan membuat kamu lebih mudah men-debug masalah di project nyata.

// Tujuan Lab
  • Memahami IoC Container dan siklus hidup Bean
  • Menggunakan @Component, @Service, @Repository, @Controller
  • Dependency Injection via constructor (best practice)
  • Menggunakan @Value dan @ConfigurationProperties
  • Memahami Scope: Singleton vs Prototype
UserService.java — Constructor Injection
java
@Service                        // Spring mengelola bean ini
@RequiredArgsConstructor        // Lombok: auto constructor untuk final fields
public class UserService {

    // ✅ Constructor injection — lebih mudah di-test
    private final UserRepository userRepository;
    private final PasswordEncoder passwordEncoder;

    public User findById(Long id) {
        return userRepository.findById(id)
            .orElseThrow(() -> new ResourceNotFoundException("User not found"));
    }
}

Lab 3 — REST API dengan Spring MVC

Membangun CRUD API yang clean — menggunakan DTO, ResponseEntity, dan konvensi REST yang benar.

// Tujuan Lab
  • Membuat REST Controller dengan @RestController dan @RequestMapping
  • Menggunakan DTO terpisah untuk request dan response
  • Mengembalikan ResponseEntity dengan HTTP status yang tepat
  • Validasi input dengan @Valid dan Bean Validation
  • Membuat API docs dengan Swagger/OpenAPI
ProductController.java
java
@RestController
@RequestMapping("/api/v1/products")
@RequiredArgsConstructor
public class ProductController {

    private final ProductService productService;

    @GetMapping
    public ResponseEntity<List<ProductResponse>> getAll() {
        return ResponseEntity.ok(productService.findAll());
    }

    @GetMapping("/{id}")
    public ResponseEntity<ProductResponse> getById(@PathVariable Long id) {
        return ResponseEntity.ok(productService.findById(id));
    }

    @PostMapping
    public ResponseEntity<ProductResponse> create(
            @Valid @RequestBody CreateProductRequest request) {
        ProductResponse created = productService.create(request);
        return ResponseEntity.status(HttpStatus.CREATED).body(created);
    }

    @PutMapping("/{id}")
    public ResponseEntity<ProductResponse> update(
            @PathVariable Long id,
            @Valid @RequestBody UpdateProductRequest request) {
        return ResponseEntity.ok(productService.update(id, request));
    }

    @DeleteMapping("/{id}")
    public ResponseEntity<Void> delete(@PathVariable Long id) {
        productService.delete(id);
        return ResponseEntity.noContent().build();
    }
}
CreateProductRequest.java — Bean Validation
java
@Data

Lab 4 — JPA, Hibernate & Database

Mapping entity ke tabel database, relasi antar tabel, dan menulis query yang efisien.

// Tujuan Lab
  • Mendefinisikan Entity dengan anotasi JPA yang tepat
  • Mapping relasi: OneToMany, ManyToOne, ManyToMany
  • Menggunakan Spring Data Repository (findBy, custom @Query)
  • Pagination dan sorting dengan Pageable
  • Menghindari N+1 Problem dengan FETCH JOIN
Product.java — JPA Entity
java
@Entity
@Table(name = "products")
@Data
@NoArgsConstructor
@AllArgsConstructor
@Builder
public class Product {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false, length = 100)
    private String name;

    @Column(nullable = false, precision = 10, scale = 2)
    private BigDecimal price;

    // ManyToOne: banyak produk milik satu kategori
    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "category_id")
    private Category category;

    // Audit fields — auto set oleh Spring Data
    @CreatedDate
    private LocalDateTime createdAt;

    @LastModifiedDate
    private LocalDateTime updatedAt;
}
ProductRepository.java — Custom Queries
java
public interface ProductRepository extends JpaRepository<Product, Long> {

    // Derived query — Spring otomatis buat SQL-nya
    List<Product> findByCategoryId(Long categoryId);
    Optional<Product> findByName(String name);
    Page<Product> findByCategoryId(Long categoryId, Pageable pageable);

    // JPQL — hindari N+1 dengan JOIN FETCH
    @Query("SELECT p FROM Product p JOIN FETCH p.category WHERE p.price BETWEEN :min AND :max")
    List<Product> findByPriceRange(@Param("min") BigDecimal min,
                                  @Param("max") BigDecimal max);

    // Native SQL query
    @Query(value = "SELECT * FROM products WHERE stock > 0 LIMIT :limit",
           nativeQuery = true)
    List<Product> findAvailableProducts(@Param("limit") int limit);
}
⚠️ N+1 Problem: Jika relasi @OneToMany di-load dengan FetchType.LAZY, akses ke collection dalam loop akan trigger query baru per item. Gunakan JOIN FETCH atau @EntityGraph untuk menghindarinya.

Lab 5 — Spring Security & JWT Authentication

Implementasi autentikasi stateless menggunakan JWT — yang paling umum digunakan untuk REST API modern.

// Tujuan Lab
  • Konfigurasi Spring Security untuk REST API (disable CSRF, stateless)
  • Implementasi UserDetailsService custom
  • Generate dan validasi JWT token
  • Membuat JwtAuthenticationFilter
  • Proteksi endpoint berdasarkan role
SecurityConfig.java
java
@Configuration
@EnableWebSecurity
@RequiredArgsConstructor
public class SecurityConfig {

    private final JwtAuthenticationFilter jwtFilter;
    private final UserDetailsService userDetailsService;

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .csrf(AbstractHttpConfigurer::disable)
            .sessionManagement(s -> s.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/v1/auth/**").permitAll()
                .requestMatchers("/swagger-ui/**", "/v3/api-docs/**").permitAll()
                .requestMatchers(HttpMethod.GET, "/api/v1/products/**").permitAll()
                .requestMatchers("/api/v1/admin/**").hasRole("ADMIN")
                .anyRequest().authenticated()
            )
            .addFilterBefore(jwtFilter, UsernamePasswordAuthenticationFilter.class);

        return http.build();
    }

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }

    @Bean
    public AuthenticationManager authManager(AuthenticationConfiguration config) throws Exception {
        return config.getAuthenticationManager();
    }
}
JwtService.java — Generate & Validate Token
java
@Service
public class JwtService {

    @Value("${app.jwt.secret}")
    private String secret;

    @Value("${app.jwt.expiration}")
    private Long expiration;

    public String generateToken(UserDetails userDetails) {
        return Jwts.builder()
            .subject(userDetails.getUsername())
            .issuedAt(new Date())
            .expiration(new Date(System.currentTimeMillis() + expiration))
            .signWith(getSignKey())
            .compact();
    }

    public String extractUsername(String token) {
        return extractClaim(token, Claims::getSubject);
    }

    public boolean isTokenValid(String token, UserDetails userDetails) {
        final String username = extractUsername(token);
        return username.equals(userDetails.getUsername()) && !isTokenExpired(token);
    }

    private SecretKey getSignKey() {
        return Keys.hmacShaKeyFor(Decoders.BASE64.decode(secret));
    }
}

Lab 6 — Global Exception Handling & Validasi

Response error yang konsisten adalah tanda API yang profesional. Semua error harus punya format yang sama dan HTTP status yang tepat.

GlobalExceptionHandler.java
java
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(ResourceNotFoundException.class)
    public ResponseEntity<ErrorResponse> handleNotFound(ResourceNotFoundException ex) {
        return ResponseEntity
            .status(HttpStatus.NOT_FOUND)
            .body(new ErrorResponse("NOT_FOUND", ex.getMessage()));
    }

    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ResponseEntity<ErrorResponse> handleValidation(MethodArgumentNotValidException ex) {
        Map<String, String> errors = new HashMap<>();
        ex.getBindingResult().getFieldErrors()
            .forEach(e -> errors.put(e.getField(), e.getDefaultMessage()));
        return ResponseEntity
            .status(HttpStatus.BAD_REQUEST)
            .body(new ErrorResponse("VALIDATION_ERROR", "Input tidak valid", errors));
    }

    @ExceptionHandler(Exception.class)
    public ResponseEntity<ErrorResponse> handleGeneral(Exception ex) {
        return ResponseEntity
            .status(HttpStatus.INTERNAL_SERVER_ERROR)
            .body(new ErrorResponse("INTERNAL_ERROR", "Terjadi kesalahan sistem"));
    }
}
ErrorResponse.java — Format error seragam
java
@Data
@AllArgsConstructor
public class ErrorResponse {

    private String message;
    private Object details;
    private LocalDateTime timestamp = LocalDateTime.now();

    public ErrorResponse(String code, String message) {
        this.code = code; this.message = message;

Lab 7 — Clean Architecture & Best Practices

Code yang mudah dibaca, di-test, dan di-maintain adalah aset utama tim engineering.

// Tujuan Lab
  • Memisahkan concern: Controller → Service → Repository
  • Menggunakan MapStruct untuk mapping Entity ↔ DTO
  • Menerapkan @Transactional dengan benar
  • Menggunakan Builder Pattern (via Lombok @Builder)
  • Logging yang informatif dengan SLF4J / Logback
ProductService.java — Transactional + Logging
java
@Service
@RequiredArgsConstructor
@Slf4j                                   // Lombok: inject log field
public class ProductService {

    private final ProductRepository repo;
    private final ProductMapper mapper;  // MapStruct

    @Transactional(readOnly = true)
    public Page<ProductResponse> findAll(Pageable pageable) {
        return repo.findAll(pageable).map(mapper::toResponse);
    }

    @Transactional
    public ProductResponse create(CreateProductRequest req) {
        log.info("Creating product: {}", req.getName());

        if (repo.findByName(req.getName()).isPresent()) {
            throw new DuplicateResourceException("Product sudah ada: " + req.getName());
        }

        Product product = mapper.toEntity(req);
        Product saved = repo.save(product);
        log.info("Product created with id: {}", saved.getId());

        return mapper.toResponse(saved);
    }
}
💡 Gunakan @Transactional(readOnly = true) untuk operasi baca. Ini meningkatkan performa karena Hibernate skip dirty checking.
🧪 Latihan Mandiri
  1. Implementasikan MapStruct mapper untuk Product ↔ ProductResponse ↔ CreateProductRequest.
  2. Review kode yang sudah dibuat di lab sebelumnya — pastikan tidak ada business logic di Controller.
  3. Tambahkan logging yang informatif di setiap method Service dengan level yang tepat (INFO untuk operasi normal, WARN untuk kondisi tidak diharapkan, ERROR untuk exception).

Lab 8 — Unit Testing & Integration Testing

Kode yang tidak di-test adalah kode yang tidak dipercaya. Minimal 1 test per service method dan 1 integration test per controller.

ProductServiceTest.java — Unit Test dengan Mockito
java
@ExtendWith(MockitoExtension.class)
class ProductServiceTest {

    @Mock
    private ProductRepository repo;

    @Mock
    private ProductMapper mapper;

    @InjectMocks
    private ProductService service;

    @Test
    void shouldReturnProductWhenFound() {
        // Arrange
        Product product = Product.builder().id(1L).name("Laptop").build();
        Mockito.when(repo.findById(1L)).thenReturn(Optional.of(product));
        Mockito.when(mapper.toResponse(product)).thenReturn(new ProductResponse(1L, "Laptop"));

        // Act
        ProductResponse result = service.findById(1L);

        // Assert
        assertThat(result.getName()).isEqualTo("Laptop");
        Mockito.verify(repo).findById(1L);
    }

    @Test
    void shouldThrowWhenProductNotFound() {
        Mockito.when(repo.findById(99L)).thenReturn(Optional.empty());
        assertThatThrownBy(() -> service.findById(99L))
            .isInstanceOf(ResourceNotFoundException.class);
    }
}
